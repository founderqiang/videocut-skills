# 静音段处理

## 阈值规则

| 静音长度 | 处理 |
|---------|------|
| ≤ 0.2秒 | **忽略** - 自然停顿 |
| 0.2-1秒 | **预选删除** - 句间卡顿 |
| > 1秒 | **建议删除** - 明显卡顿或展示画面 |

## 输出格式

**整段标记，不拆分**

示例：3.2 秒静音 → 输出 1 条
```
| 64-66 | 12.86-15.80 | 静音3.2s | | 删 |
```

用户在审核网页中取消不想删的。

## 特殊情况

### 长静音
连续 5s+ 的静音整段标记，预选删除：
```
| 323-371 | 71.38-131.38 | 静音60s | | 删 |
```

### 开头静音
视频开头的静音必删：
```
| 0 | 0.00-1.00 | 静音1s | 开头静音 | 删 |
```

### 🚨 头尾未转录裁剪（转录盲区，必做）

**火山只转录语音。** 结尾常有**未转录的杂音/收尾动作**（说完的"好"、停顿响声、伸手关录制），它**不在 subtitles_words.json 里 → 没有 idx → 任何检测器都看不见 → 网页上选不了 → 成片会带着这段杂音**。

必做：比对**视频真实时长 vs 转录首尾**。

| 位置 | 判定 | 处理 |
|------|------|------|
| 结尾 | `视频时长 − 最后一个字.end > 0.3s` | 补一个尾元素 `[last.end → 视频时长]`，预选删除 |
| 开头 | `第一个元素.start > 0.3s` | 通常 w[0] 是 0 起的 gap 已覆盖；若没有则补头元素（注意补头会移位 idx） |

判断头尾是杂音还是静音都无所谓——**都该删**。实测一次结尾 4.54s 漏剪：max -16.8dB 的真响声，纯靠转录完全看不见（见 [log/精简版方法论-风险分层与口播稿对齐.md] 同期发现）。执行见 SKILL.md 步骤 5.4b。
